# ========== 1. AI产品核心规则 ==========


1. 优先锚定「用户真实痛点」，所有AI功能设计拒绝“为AI而AI”，必须明确AI的核心价值的落地场景；

2. 严格遵循AI合规规范（《生成式人工智能服务管理暂行办法》《个人信息保护法》），数据采集、使用、存储全程合规；

3. 明确主进程（PM核心决策）、辅助进程（AI功能支撑）分工：AI仅提供参考，不替代PM做最终决策；

4. AI需求必须量化目标（准确率、使用率、问题解决率等），拒绝模糊表述，所有方案需有数据支撑；

5. AI功能采用“小步快跑”模式，优先落地MVP版本，通过灰度验证价值后再迭代优化，规避技术风险。

# ========== 2. 需求定义阶段规范 ==========


1. 精准拆解AI能力边界，明确「AI能做」（海量数据分析、重复工作自动化、个性化推荐）与「AI不该做」（复杂决策、情感共情、非标准化场景）；

2. 需求调研需聚焦“AI使用场景+用户接受度”，避免技术驱动忽略用户习惯，输出场景化需求文档；

3. AI需求需明确优先级，按“投入产出比+用户痛点强度”排序，优先落地高价值、低难度的AI功能；

4. 需求文档需标注AI相关核心指标，明确验收标准，避免模糊化表述（禁用“让AI更智能”等无效描述）；

5. 需求评审需同步AI技术边界，与算法团队确认可行性，提前规避“需求无法落地”的问题。

# ========== 3. 产品设计阶段规范 ==========


1. 交互设计：AI入口清晰，标注“AI辅助/AI生成”标识，降低用户认知成本，提供一键关闭/纠错入口；

2. 功能设计：同一产品AI功能聚焦1-2个核心场景，不堆砌功能，明确AI与人工的分工边界；

3. 数据设计：仅采集AI必需数据，明确告知用户数据用途并获取授权，敏感数据加密脱敏，不滥用用户数据；

4. 容错设计：AI输出错误时，提供简单纠错入口，同步记录错误数据，用于后续模型优化；

5. 适配设计：结合用户画像调整AI输出，专业场景输出详细数据，普通场景输出简化结论，提升适配性。

# ========== 4. 研发落地阶段规范 ==========


1. 跨团队协同：与算法/测试团队明确分工，清晰传递AI需求与指标，定期同步进度，解决技术瓶颈；

2. 技术选型：按需求适配AI方案（轻量需求用第三方API，核心需求自研模型），平衡效果与研发成本；

3. 测试规范：重点测试AI准确率、边界场景（极端输入/错误输入）、安全合规性，不遗漏核心测试点；

4. 灰度发布：AI功能先面向小部分用户开放，收集反馈优化模型，再逐步扩大覆盖范围；

5. 应急机制：灰度/上线期间，若出现AI输出错误、安全问题，可快速回滚版本，降低用户影响。

# ========== 5. 迭代优化阶段规范 ==========


1. 数据监测：建立AI核心指标监测体系，实时跟踪准确率、使用率、纠错率，及时发现性能下降问题；

2. 模型优化：定期收集用户纠错、人工校验数据，更新训练数据集，优化模型参数，提升输出质量；

3. 需求迭代：基于用户反馈与数据，动态调整AI功能，使用率低、无价值的功能及时暂停迭代；

4. 风险复盘：每轮迭代后，复盘AI合规风险、技术风险，形成优化方案，避免重复问题；

5. 行业同步：关注AI技术与合规动态，适时引入新方案，保持产品竞争力，贴合行业基准。

1. 输出的AI产品方案、规则文档需完整可落地，包含必要的依赖说明、指标定义、实施步骤；

2. 明确兼容性说明（如AI模型适配的技术环境、合规要求适配的地区/行业）；

3. 指出潜在问题：如AI伦理风险、数据安全隐患、技术落地难点，同步给出规避方案；

4. 文档排版简洁规范，按“模块→要点”分级呈现，关键规则加粗标注，便于快速查阅；

5. 所有交付内容需符合职场规范，语言专业简洁，拒绝口语化，可直接用于团队协作与落地执行。
