## 一、项目概述

### 1.1 项目定位

**Kim 多级分镜提示词助手**是一款面向视频创作者和 AI 内容生成师的桌面应用，提供**多级分镜管理**与**AI 提示词自动化生成**功能。

### 1.2 核心目标

| 目标 | 说明 |
|------|------|
| **结构化分镜管理** | 项目 → 片段 → 镜头 三层结构化管理，支持属性编辑和素材关联 |
| **AI 提示词生成** | 集成 DeepSeek、豆包、通义千问等大模型，自动生成视频分镜脚本 |
| **模板化创作** | 支持自定义分镜模板，快速生成标准化 JSON 数据 |
| **素材资源管理** | 项目级和片段级双素材库，支持图片/视频/音频分类存储 |
| **本地化存储** | 数据本地存储，无需依赖云端服务，保障创作隐私 |

### 1.3 目标用户

- **视频创作者**：短视频、宣传片、广告视频的分镜脚本创作
- **AI 内容生成师**：使用 AI 工具生成视频内容的专业人员
- **影视前期制作**：分镜师、导演、编剧的可视化脚本工具

### 4.5 关键技术实现

#### 4.5.1 Sandbox 模式文件上传

**问题**: Sandbox 模式下无法获取 File 对象的真实路径

**解决方案**: 使用 FileReader 读取为 Base64，通过 IPC 传递给主进程保存

```javascript
// 渲染进程：读取文件为 Base64
function readFileAsBase64(file) {
  return new Promise((resolve, reject) => {
    const reader = new FileReader();
    reader.readAsDataURL(file);
    reader.onload = () => resolve(reader.result);
    reader.onerror = reject;
  });
}

// 主进程：保存 Base64 文件
ipcMain.handle('fs:saveDroppedFile', async (event, fileName, fileData, projectDir, assetType) => {
  const matches = fileData.match(/^data:(.+);base64,(.+)$/);
  const buffer = Buffer.from(matches[2], 'base64');
  fs.writeFileSync(targetPath, buffer);
});
```

#### 4.5.2 视频缩略图提取

**问题**: 视频直接显示首帧可能是黑场

**解决方案**: 动态跳转检测亮度，自动寻找有效帧

```javascript
function extractVideoFrame(video) {
  let seekCount = 0;
  const maxSeeks = 2;
  
  video.onseeked = () => {
    const brightness = detectBrightness(canvas);
    if (brightness < 30 && seekCount < maxSeeks) {
      // 黑场，跳转到下一时间点
      video.currentTime += 0.5;
      seekCount++;
    } else {
      // 有效帧，替换为 img 元素
      replaceWithImage(video, canvas);
    }
  };
}
```
