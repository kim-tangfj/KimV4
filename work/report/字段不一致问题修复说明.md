# 字段不一致问题修复说明

## 问题原因

现有项目文件 (`project.json`) 中的字段与新开发的属性表单字段不一致：

### 旧字段结构
```json
{
  "shots": [{
    "name": "片段名称",
    "description": "片段描述",
    "duration": 10,
    "style": "风格",
    "mood": "情绪",
    "scenes": [{
      "name": "镜头名称",
      "shotType": "景别",
      "angle": "角度",
      "camera": "运镜",
      "content": "内容",
      "duration": 5,
      "dialogue": "台词",
      "emotion": "情绪",
      "notes": "备注"
    }]
  }]
}
```

### 新字段结构
```json
{
  "shots": [{
    "name": "片段名称",
    "description": "片段描述",
    "duration": 10,
    "style": "风格",
    "mood": "情绪氛围",
    "characters": "角色",           // 新增
    "sceneSetting": "场景设定",      // 新增
    "musicStyle": "配乐风格",        // 新增
    "soundEffect": "音效需求",       // 新增
    "imageRef": "图片参考",          // 新增
    "videoRef": "视频参考",          // 新增
    "audioRef": "音频参考",          // 新增
    "customPrompt": "补充提示词",    // 新增
    "scenes": [{
      "name": "镜头名称",
      "shotType": "景别",
      "angle": "角度",
      "camera": "运镜",
      "image": "分镜图片",           // 新增
      "content": "内容描述",
      "duration": 5,
      "dialogue": "对白内容",
      "emotion": "情绪描述",
      "notes": "其他备注"
    }]
  }]
}
```

## 已修复内容

### 1. 更新默认模板 (`assets/default/default-templates.json`)
- 添加新字段到 AI 生成模板
- 确保 AI 生成的数据包含所有新字段

### 2. 字段名称统一
**片段属性**:
- HTML 表单 ID: `shotCharacters`, `shotSceneSetting`, `shotMusicStyle`, `shotSoundEffect`, `shotImageRef`, `shotVideoRef`, `shotAudioRef`, `shotCustomPrompt`
- 保存函数变量：`characters`, `sceneSetting`, `musicStyle`, `soundEffect`, `imageRef`, `videoRef`, `audioRef`, `customPrompt`
- JSON 字段名：`characters`, `sceneSetting`, `musicStyle`, `soundEffect`, `imageRef`, `videoRef`, `audioRef`, `customPrompt`

**镜头属性**:
- HTML 表单 ID: `sceneImage`, `sceneContent`, `sceneDialogue`
- 保存函数变量：`image`, `content`, `dialogue`
- JSON 字段名：`image`, `content`, `dialogue`

### 3. 自动保存功能
- 失焦后 500ms 自动保存
- 保存时检查选项变更并增加使用次数
- 保存后更新 `appState` 状态

## 测试步骤

### 方法 1: 创建新项目（推荐）
1. 重启应用
2. 新建项目（使用 AI 生成或手动创建）
3. 选中一个片段
4. 修改属性表单中的字段
5. 点击其他片段或等待 500ms
6. 检查 `work/` 目录下的项目文件是否保存成功

### 方法 2: 更新现有项目
1. 打开现有项目
2. 选中一个片段
3. **手动填写所有新字段**（角色、场景设定、配乐风格等）
4. 切换到另一个片段
5. 再次切回，检查字段是否保留
6. 重启应用，检查字段是否保留

### 方法 3: 手动修改项目文件
1. 打开 `work/<项目名>/project.json`
2. 为每个 shot 添加新字段：
```json
{
  "shots": [{
    "id": "...",
    "name": "...",
    "characters": "",
    "sceneSetting": "",
    "musicStyle": "",
    "soundEffect": "",
    "imageRef": "",
    "videoRef": "",
    "audioRef": "",
    "customPrompt": "",
    "scenes": [{
      "id": "...",
      "name": "...",
      "image": "",
      "content": "...",
      "dialogue": "..."
    }]
  }]
}
```
3. 保存文件
4. 重启应用

## 注意事项

1. **旧项目数据兼容**: 新字段使用 `|| ''` 默认值，旧项目数据不会报错，但需要手动填写新字段

2. **自动保存触发条件**:
   - 输入框失焦（blur 事件）
   - 选项下拉框变更（change 事件）
   - 500ms 防抖延迟

3. **检查保存成功**:
   - 右下角显示"已保存"提示
   - 检查 `work/<项目名>/project.json` 文件修改时间
   - 切换片段后返回，检查字段值是否保留

## 常见问题

### Q: 修改属性后没有保存？
A: 检查：
1. 是否在 Electron 环境中运行（`useElectronAPI` 标志）
2. `appState.currentProject.projectDir` 是否有值
3. 控制台是否有错误信息

### Q: 新字段显示为空？
A: 正常现象，旧项目数据没有这些字段。手动填写后会自动保存。

### Q: 选项字段（风格、情绪等）不显示？
A: 检查：
1. 自定义选项管理中有无对应组别的选项
2. `loadOptionsByGroup()` 函数是否正常加载
3. 控制台有无错误信息
