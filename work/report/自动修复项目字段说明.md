# 项目字段自动修复工具

## 功能说明
自动读取现有 `project.json` 文件，添加缺失的新字段，避免手动修改。

## 新增字段

### 片段字段
- `characters` - 角色描述
- `sceneSetting` - 场景设定
- `musicStyle` - 配乐风格
- `soundEffect` - 音效需求
- `imageRef` - 图片参考
- `videoRef` - 视频参考
- `audioRef` - 音频参考
- `customPrompt` - 补充提示词

### 镜头字段
- `image` - 分镜图片

## 备份说明

脚本会自动创建备份文件：
- 原文件：`project.json`
- 备份文件：`project.json.backup`

**恢复方法**: 删除 `project.json`，重命名 `project.json.backup` 为 `project.json`

## 查找项目文件

脚本会自动查找以下位置的项目：
1. `e:\AI\KimV4\projects\<项目名>\project.json`
2. `e:\AI\KimV4\work\<项目名>\project.json`

## 手动指定项目文件

如果脚本没有找到项目文件，可以修改脚本中的路径：

```javascript
// 修改第 12 行
const PROJECTS_DIR = path.join(WORK_DIR, 'work'); // 改为实际的项目目录
```

## 验证修复结果

1. **重启应用**
2. **打开项目**
3. **选中片段**，检查属性表单：
   - 角色字段是否有值
   - 场景设定字段是否有值
   - 配乐风格、音效需求字段是否可编辑
4. **修改字段**，切换到另一个片段，再切回来检查是否保存成功

## 常见问题

### Q: 脚本报错 "找不到模块"？
A: 确保已安装 Node.js，并在项目目录运行：
```powershell
npm install
```

### Q: 脚本运行后没有反应？
A: 检查：
1. 项目文件路径是否正确
2. `project.json` 是否是有效的 JSON 格式
3. PowerShell 是否有执行权限

### Q: 修复后字段还是空的？
A: 正常现象。脚本只添加字段结构（空值），需要手动填写内容。

### Q: 可以只修复特定项目吗？
A: 可以。修改脚本中的 `findProjectFiles()` 函数，只返回需要修复的项目文件路径。

## 批量修复多个项目

脚本默认会修复所有找到的项目。如果有大量项目，建议：
1. 先修复 1 个项目测试
2. 确认无误后批量修复
3. 保留所有 `.backup` 文件，以备不时之需

## 还原修改

如需还原到修复前的状态：
```powershell
# PowerShell 命令
cd e:\AI\KimV4\work\<项目名>
Move-Item -Force project.json.backup project.json
```
