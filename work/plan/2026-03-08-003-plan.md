## 一、功能概述

### 1.1 功能说明

素材引用功能允许用户在镜头描述中快速引用项目素材库中的图片、视频、音频等素材，无需手动复制粘贴长路径。

### 1.2 使用场景

```
场景 1：描述镜头画面
┌─────────────────────────────────────────────────────────┐
│  镜头描述：主角站在 @森林背景 前，表情坚定               │
│           ↑                                            │
│           └── 使用 @素材名 快速引用                      │
└─────────────────────────────────────────────────────────┘

场景 2：添加背景音乐
┌─────────────────────────────────────────────────────────┐
│  镜头描述：清晨的森林，阳光透过树叶                      │
│  背景音乐：@鸟鸣声.mp3                                   │
│           ↑                                            │
│           └── 引用音频素材                               │
└─────────────────────────────────────────────────────────┘

场景 3：参考角色设定
┌─────────────────────────────────────────────────────────┐
│  镜头描述：按照 @团团角色设定 绘制小刺猬                 │
│           ↑                                            │
│           └── 引用角色设定图                             │
└─────────────────────────────────────────────────────────┘
```

  ]

## 三、技术实现

### 3.1 数据结构

#### 镜头数据结构扩展
```json
{
  "id": "scene_001",
  "name": "森林中的团团",
  "content": "主角 @团团角色 站在 @森林背景 前",
  "duration": 5,
  "references": [
    {
      "id": "ref_001",
      "name": "团团角色",
      "path": "assets/images/团团角色.png",
      "type": "image",
      "insertedAt": 12
    },
    {
      "id": "ref_002",
      "name": "森林背景",
      "path": "assets/images/森林背景.jpg",
      "type": "image",
      "insertedAt": 20
    }
  ]
}
```

#### 引用解析器
```javascript
/**
 * 解析镜头描述中的素材引用
 * @param {string} content - 镜头描述
 * @param {Object} projectData - 项目数据
 * @returns {Object} 解析结果
 */
function parseReferences(content, projectData) {
  const references = [];
  const regex = /@([^@\s\[\]]+)/g;
  let match;

  while ((match = regex.exec(content)) !== null) {
    const素材名 = match[1];
    const 素材 = findAssetByName(projectData.assets, 素材名);
    
    if (素材) {
      references.push({
        name: 素材名，
        path: 素材.path,
        type: 素材.type,
        position: match.index
      });
    }
  }

  return { content, references };
}

/**
 * 生成 AI 提示词时替换引用
 * @param {string} content - 镜头描述
 * @param {Array} references - 引用列表
 * @returns {string} 替换后的提示词
 */
function generatePrompt(content, references) {
  let prompt = content;
  
  // 替换 @素材名 为具体描述
  references.forEach(ref => {
    const 描述 = getAssetDescription(ref);
    prompt = prompt.replace(`@${ref.name}`, 描述);
  });
  
  return prompt;
}
```

### 3.3 提示词生成集成

#### 当前提示词生成流程
```
项目 → 片段 → 镜头 → 提示词生成
                      ↓
                  读取镜头 content
                      ↓
                  替换 @引用
                      ↓
                  发送给 LLM
```

#### 修改点
```javascript
// promptGenerator.js
async function generateShotPrompt(shot, projectData) {
  // 获取镜头描述
  const content = scene.content;
  
  // 解析引用
  const { references } = parseReferences(content, projectData);
  
  // 替换引用为描述
  const prompt = generatePrompt(content, references);
  
  // 发送给 LLM
  return await callLlmApi(prompt);
}
```

## 七、注意事项

### 7.1 数据兼容

- 现有镜头数据不包含 `references` 字段，需要向后兼容
- 旧项目的 `@引用` 语法需要能正常解析

### 7.2 用户体验

- @触发选择器不应干扰正常输入
- 拖拽插入应有视觉反馈
- 引用素材应有明显标识

### 7.3 性能优化

- 素材选择器应支持搜索和过滤
- 大量素材时应使用虚拟滚动
- 引用解析应缓存结果
