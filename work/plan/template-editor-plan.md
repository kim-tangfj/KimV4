# 模板编辑器开发计划

**版本**: v1.2.0  
**预计工时**: 12 小时  
**优先级**: P1  
**计划日期**: 2026-03-24 ~ 2026-03-31

## 一、需求分析

### 1.1 当前模板系统状态

#### 已有功能 ✅
| 功能 | 状态 | 说明 |
|------|------|------|
| 模板加载/保存 | ✅ | 通过 IPC 调用主进程读写 JSON |
| 模板备份/恢复 | ✅ | 导出/导入 JSON 文件 |
| 模板列表展示 | ✅ | 简单的列表 + 激活/编辑/删除按钮 |
| 基础编辑器 | ✅ | 纯文本 textarea 编辑模板内容 |
| 5 个默认模板 | ✅ | 包含分镜/广告/教程/手绘动画模板 |

#### 存在的问题 ❌
| 问题 | 影响 |
|------|------|
| 编辑器是纯文本框 | 无语法高亮，无结构提示 |
| 无模板预览 | 无法直观看到模板效果 |
| 无变量插入辅助 | 需要手动输入 `{剧本内容}` 等占位符 |
| 无模板分类 | 所有模板混在一起 |
| 无模板导入/导出 | 无法分享自定义模板 |
| 无模板测试 | 无法快速验证模板生成效果 |

### 1.2 用户需求场景

| 场景 | 需求 |
|------|------|
| 新手用户 | 需要模板示例和引导，快速上手 |
| 进阶用户 | 需要自定义模板结构，调整提示词 |
| 专业用户 | 需要多模板管理，分类筛选 |
| 分享需求 | 需要导出模板分享给其他人 |

## 二、功能设计

### 2.1 功能架构图

```
模板编辑器 v2.0
├── 模板库管理
│   ├── 模板列表（卡片视图）
│   ├── 模板分类筛选
│   ├── 模板搜索
│   └── 模板排序
│
├── 可视化编辑器
│   ├── 基础信息编辑（名称/描述/分类）
│   ├── 提示词编辑器（语法高亮）
│   ├── 变量插入助手
│   ├── 实时预览
│   └── 模板测试
│
├── 模板导入/导出
│   ├── 导出为 JSON 文件
│   ├── 导入 JSON 模板
│   └── 导入时冲突处理
│
└── 模板预设
    ├── 新建空白模板
    ├── 从现有模板复制
    └── 使用预设模板
```

### 2.2 详细功能清单

#### 功能 1: 模板库管理（2h）

**1.1 卡片视图**
- 将列表视图改为卡片网格布局
- 每张卡片显示：模板名称、描述、分类标签、状态（已激活/未激活）
- 卡片操作：激活、编辑、删除、导出

**1.2 分类筛选**
- 新增模板分类字段：`category`
- 预设分类：分镜脚本、商品广告、教程教学、动画、其他
- 筛选器：全部分类、分镜脚本、商品广告、教程教学、动画、其他

**1.3 搜索功能**
- 支持按名称/描述搜索
- 实时过滤模板列表

**1.4 排序功能**
- 按名称排序（A-Z）
- 按创建时间排序
- 按使用时间排序

#### 功能 2: 可视化编辑器（6h）

**2.1 基础信息编辑**
```
┌─────────────────────────────────────┐
│ 模板名称：[___________________]     │
│ 模板描述：[___________________]     │
│             [___________________]   │
│ 模板分类：[分镜脚本 ▼]              │
│ 默认时长：[15 ▼] 秒                 │
│ 默认画幅：[16:9 ▼]                  │
└─────────────────────────────────────┘
```

**2.2 提示词编辑器（核心功能）**
- 使用 CodeMirror 或 Monaco Editor 实现语法高亮
- 支持的语言模式：Markdown + JSON 混合
- 语法高亮规则：
  - 占位符 `{剧本内容}` 显示为橙色
  - JSON 键名显示为蓝色
  - JSON 字符串显示为绿色
  - 注释显示为灰色

**2.3 变量插入助手**
```
┌─────────────────────────────────────┐
│ 插入变量：                          │
│ [ {剧本内容} ] [ {角色} ] [ {场景} ]│
│ [ {风格} ] [ {情绪} ] [ {时长} ]    │
│ [ {画幅} ] [ {对白} ] [ {备注} ]    │
└─────────────────────────────────────┘
```

**2.4 实时预览**
- 右侧分栏显示渲染后的提示词效果
- 占位符高亮显示
- 支持一键复制预览内容

**2.5 模板测试**
- 提供测试输入框（输入示例剧本）
- 调用 LLM API 测试模板生成效果
- 显示生成的 JSON 结果

#### 功能 3: 模板导入/导出（2h）

**3.1 导出功能**
- 单个导出：右键菜单/卡片操作 → 导出
- 批量导出：选择多个模板 → 批量导出为 ZIP
- 导出文件格式：`模板名称.json`

**3.2 导入功能**
- 拖放导入：拖拽 JSON 文件到模板库
- 文件选择导入：选择 JSON 文件
- 冲突处理：
  - 跳过（保留原模板）
  - 覆盖（替换原模板）
  - 重命名（保留两者）

**3.3 导入验证**
- 验证 JSON 格式
- 验证必填字段（id, name, content）
- 验证占位符是否存在

#### 功能 4: 模板预设（2h）

**4.1 新建模板向导**
```
┌─────────────────────────────────────┐
│ 新建模板                            │
│                                     │
│ ○ 空白模板                          │
│ ○ 从现有模板复制                    │
│ ○ 使用预设模板                      │
│                                     │
│ 预设模板列表：                      │
│ - 默认分镜模板（15s）               │
│ - 默认分镜模板（8s）                │
│ - 商品广告模板                      │
│ - 教程教学模板                      │
│ - 手绘动画模板                      │
└─────────────────────────────────────┘
```

**4.2 模板复制**
- 选择源模板
- 输入新模板名称
- 自动修改 ID（添加时间戳）

## 三、技术实现方案

### 3.1 数据结构更新

#### 当前模板结构
```json
{
  "id": "default_storyboard_15",
  "name": "默认分镜模板（15s）",
  "description": "标准的视频分镜脚本模板",
  "content": "提示词内容..."
}
```

#### 新模板结构
```json
{
  "id": "default_storyboard_15",
  "name": "默认分镜模板（15s）",
  "description": "标准的视频分镜脚本模板，适用于大多数视频创作场景",
  "category": "storyboard",
  "content": "提示词内容...",
  "variables": ["剧本内容", "角色", "场景"],
  "settings": {
    "defaultDuration": 15,
    "defaultAspectRatio": "16:9"
  },
  "metadata": {
    "createdAt": "2026-03-10T00:00:00.000Z",
    "updatedAt": "2026-03-10T00:00:00.000Z",
    "usageCount": 10,
    "isBuiltIn": true
  }
}
```

### 3.2 新增 IPC 接口

```javascript
// preload.js
{
  // 模板导入/导出
  exportTemplate: (templateId) => ipcRenderer.invoke('template:export', templateId),
  exportTemplatesBatch: (templateIds) => ipcRenderer.invoke('template:exportBatch', templateIds),
  importTemplate: (filePath) => ipcRenderer.invoke('template:import', filePath),
  
  // 模板测试
  testTemplate: (templateContent, testScript) => ipcRenderer.invoke('template:test', templateContent, testScript),
  
  // 获取分类列表
  getTemplateCategories: () => ipcRenderer.invoke('template:getCategories')
}
```

### 3.3 新增文件

| 文件 | 用途 | 预计行数 |
|------|------|----------|
| `src/utils/templateEditor.js` | 可视化编辑器逻辑 | ~500 |
| `src/utils/templateImport.js` | 导入/导出逻辑 | ~200 |
| `src/styles/template-editor.css` | 编辑器样式 | ~300 |
| `assets/templates/presets/` | 预设模板目录 | - |

### 3.4 依赖安装

```bash
# 代码编辑器组件（二选一）
npm install codemirror          # 轻量级，~200KB
# 或
npm install monaco-editor       # VS Code 同款，~2MB

# ZIP 支持（批量导出）
npm install jszip
```

**推荐**: 使用 CodeMirror 6（轻量、够用）

## 八、风险与应对

### 风险 1: CodeMirror 集成问题
**风险**: CodeMirror 与 Electron 兼容性问题  
**应对**: 提前调研，准备备用方案（Monaco Editor 或简单高亮）

### 风险 2: 工期延误
**风险**: 12 小时工时可能不足  
**应对**: 优先实现核心功能（编辑器 + 导入导出），高级功能（测试、预设）可延后

### 风险 3: 数据迁移
**风险**: 旧模板数据结构不兼容  
**应对**: 编写数据迁移脚本，自动升级旧模板
