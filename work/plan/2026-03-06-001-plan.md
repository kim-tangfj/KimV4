## 一、项目核心定位

**产品定位**：面向视频创作者、AI 内容生成师的多级分镜管理与提示词自动化生成工具。

**核心价值**：四级横向层级管理（项目→片段→镜头）+ 自动化提示词生成 + 本地化文件安全存储 + 底部显隐式属性操作。

## 三、细化功能点补充

### 3.1 项目管理细化

#### 3.1.1 项目创建流程
```
用户操作 → 点击 + 按钮
  ↓
弹出表单 → 输入名称、描述、选择画幅
  ↓
数据验证 → 检查名称是否为空
  ↓
创建文件夹 → 项目名称_时间戳/assets/{images,videos,audios}
  ↓
生成 project.json → 包含基础字段 + 默认测试数据
  ↓
刷新列表 → 新项目自动选中
```

#### 3.1.2 项目右键菜单
- **修改项目**：弹出表单编辑名称、描述、画幅（待实现）
- **删除项目**：二次确认 → 删除文件夹 → 刷新列表
- **打开资源文件管理器**：调用 shell.openPath 打开项目目录

#### 3.1.3 项目状态流转
```
草稿 (draft) → 进行中 (processing) → 已完成 (completed) → 已归档 (archived)
```

## 四、素材库方案策划

### 4.1 素材库架构设计

根据需求，素材库应分为两级：
1. **项目级素材库** - 整个项目共享的素材
2. **片段级素材库** - 每个片段独立的素材

#### 4.1.1 目录结构
```
项目名称_时间戳/
├── project.json              # 项目数据文件
├── assets/                   # 项目级素材库
│   ├── images/               # 项目共享图片
│   │   ├── logo.png
│   │   ├── reference_1.jpg
│   │   └── ...
│   ├── videos/               # 项目共享视频
│   │   ├── demo.mp4
│   │   └── ...
│   └── audios/               # 项目共享音频
│       ├── bgm.mp3
│       └── ...
└── shots/                    # 片段级素材库（可选）
    └── shot_1772760068170/   # 片段 ID 命名的文件夹
        ├── images/           # 该片段专属图片
        │   └── storyboard_1.png
        └── videos/           # 该片段专属视频
            └── reference.mp4
```

#### 4.1.2 数据结构设计

**project.json 中的素材引用**：
```json
{
  "project": {
    "rootAssetsDir": "F:\\AI\\KimV4\\projects\\项目名称\\assets"
  },
  "shots": [
    {
      "id": "shot_1772760068170_0",
      "name": "片段 1",
      "assets": {
        "referenceImages": [
          "assets/images/reference_1.jpg",
          "assets/images/reference_2.jpg"
        ],
        "referenceVideos": [
          "assets/videos/demo.mp4"
        ],
        "referenceAudios": [
          "assets/audio/bgm.mp3"
        ]
      },
      "scenes": [
        {
          "id": "scene_1772760068170_0_0",
          "storyboard": "shots/shot_1772760068170/images/storyboard_1.png",
          "assets": {
            "referenceImages": [
              "shots/shot_1772760068170/images/ref_1.jpg"
            ]
          }
        }
      ]
    }
  ]
}
```

### 4.2 素材库功能设计

#### 4.2.1 项目级素材库功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 素材上传 | 支持拖放/选择文件上传 | P0 |
| 素材预览 | 图片缩略图、视频播放器、音频播放器 | P0 |
| 素材分类 | 自动按 images/videos/audios 分类显示 | P0 |
| 素材引用 | 在属性面板中@引用素材 | P1 |
| 素材删除 | 删除未使用的素材 | P1 |
| 素材搜索 | 按名称搜索素材 | P2 |

#### 4.2.2 片段级素材库功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 分镜图上传 | 支持多张图片（≤9 张） | P0 |
| 分镜图预览 | 缩略图网格展示 | P0 |
| 参考素材 | 片段专属参考图片/视频 | P1 |
| 镜头参考 | 每个镜头独立的参考素材 | P2 |

### 4.3 素材引用方案

#### 4.3.1 引用格式
```
@图片 1 作为首帧，@图片 2 为角色形象
@视频 1 作为参考运镜
@音频 1 作为整体配乐
```

#### 4.3.2 素材选择器 UI
```
┌─────────────────────────────────┐
│ 图片参考（≤9 张）               │
├─────────────────────────────────┤
│ [上传图片] [从素材库选择]        │
├─────────────────────────────────┤
│ ┌───┐ ┌───┐ ┌───┐              │
│ │ 📷│ │ 📷│ │ + │  已选 2/9    │
│ └───┘ └───┘ └───┘              │
│ 图片 1  图片 2  添加             │
└─────────────────────────────────┘
```

### 4.4 素材管理建议

#### 4.4.1 推荐方案
1. **统一使用项目级素材库** - 简化结构，所有素材放在项目 assets 目录
2. **片段专属素材用子文件夹** - `assets/shots/{shotId}/`
3. **素材引用使用相对路径** - 便于项目迁移

#### 4.4.2 限制建议
- 图片：单张≤10MB，总数≤50 张/项目
- 视频：单个≤100MB，总时长≤15 秒/参考项
- 音频：单个≤50MB，总时长≤15 秒/参考项
