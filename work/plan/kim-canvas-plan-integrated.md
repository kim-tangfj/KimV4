## 一、需求分析

### 1.1 产品定位

基于现有 **Kim 多级分镜提示词助手** 项目，新增 **Kim 画布** 功能模块，为每个片段提供可视化创作画布。

**核心特点**：
- ✅ 以 **片段（Shot）** 为基础单位
- ✅ 复用 **项目素材库** 和 **片段素材库**
- ✅ 原生菜单添加 **"Kim 画布"** 入口
- ✅ 支持文生图、图生图 AI 创作

### 1.2 与现有项目的关系

```
项目 (Project)
├── 片段 1 (Shot) ──→ Kim 画布 (Canvas)
│   ├── 片段素材库
│   └── 镜头 1, 镜头 2, ...
├── 片段 2 (Shot) ──→ Kim 画布 (Canvas)
│   ├── 片段素材库
│   └── 镜头 1, 镜头 2, ...
└── 项目素材库 (所有片段共享)
```

### 1.3 核心功能

| 功能模块 | 子功能 | 优先级 |
|----------|--------|--------|
| **画布入口** | 原生菜单"Kim 画布"、片段右键菜单 | P0 |
| **素材选择** | 项目素材库、片段素材库 | P0 |
| **画布编辑** | 拖放/缩放/移动/删除/图层 | P0 |
| **文生图** | 提示词→生成→添加到画布 | P0 |
| **图生图** | 参考图 + 提示词→生成 | P1 |
| **图像编辑** | 局部重绘/扩图 | P1 |
| **保存同步** | 画布数据保存到片段 | P0 |

## 五、开发任务分解

| 1.1 添加原生菜单项 | 1h | "Kim 画布"菜单 + 快捷键 |

| 1.4 片段右键菜单入口 | 1h | 右键菜单添加入口 |

| 任务 | 工时 | 说明 |
|------|------|------|
| 2.1 画布 HTML/CSS 布局 | 2h | 画布区域、工具栏 |
| 2.2 Fabric.js 集成 | 3h | 画布引擎初始化 |
| 2.3 基础工具实现 | 3h | 选择/移动/缩放/删除 |
| 2.4 图层管理面板 | 2h | 图层列表/顺序/可见性 |

### 阶段 3: 素材库整合（8h）

| 任务 | 工时 | 说明 |
|------|------|------|
| 3.1 素材选择器 UI | 2h | 切换项目/片段素材库 |
| 3.2 素材缩略图展示 | 2h | 复用现有素材库组件 |
| 3.3 素材拖放到画布 | 3h | 拖放事件 + 图片加载 |
| 3.4 素材引用关系保存 | 1h | 保存 assetId 和 source |

### 阶段 4: AI 创作功能（12h）

| 任务 | 工时 | 说明 |
|------|------|------|
| 4.1 文生图 UI | 2h | 提示词输入/参数设置 |
| 4.2 文生图 API 调用 | 3h | 阿里百炼 API 集成 |

| 4.4 图生图 UI | 2h | 参考图选择/强度调节 |
| 4.5 图生图 API 调用 | 2h | 阿里百炼 API 集成 |

| 任务 | 工时 | 说明 |
|------|------|------|
| 5.1 画布数据保存到片段 | 2h | 更新 shot.canvas 字段 |

## 六、技术架构

### 6.1 文件结构

```
src/
├── utils/
│   ├── canvas/

│   │   ├── canvasRenderer.js     # 画布渲染

│   ├── ai-studio/
│   │   ├── textToImage.js        # 文生图
│   │   ├── imageToImage.js       # 图生图
│   │   └── imageEdit.js          # 图像编辑
│   │
│   └── assets/
│       ├── assetPicker.js        # 素材选择器
│       └── assetDragDrop.js      # 素材拖放
│
├── handlers/
```
